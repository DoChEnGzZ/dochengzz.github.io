232. 用栈实现队列 - 力扣（LeetCode）

请你仅使用两个栈实现先入先出队列。队列应当支持一般队列支持的所有操作（push、pop、peek、empty）：

实现 MyQueue 类：

void push(int x) 将元素 x 推到队列的末尾
int pop() 从队列的开头移除并返回元素
int peek() 返回队列开头的元素
boolean empty() 如果队列为空，返回 true ；否则，返回 false
说明：

你 只能 使用标准的栈操作 —— 也就是只有 push to top, peek/pop from top, size, 和 is empty 操作是合法的。
你所使用的语言也许不支持栈。你可以使用 list 或者 deque（双端队列）来模拟一个栈，只要是标准的栈操作即可。

easy题目，真的非常简单，用两个栈实现队列。栈和队列的最大区别就是先进后出和先进先出的区别。但是当我们使用连个栈，一个是入栈一个是出栈。当把第一个入栈中的元素一个一个弹出并且放入第二个栈时，栈中的元素就调了个个，先进后出就变成了先进后出。

同样的，我们该如何用两个队列实现一个栈呢？这有点不一样，仔细思考了一下发现我们不能简单地用类似的方法。因为两个先进先出的队列不能简单的让元素变成先进后出。我们只需要利用两个队列，一个做辅助队列，让入队列的顺序颠倒，队列尾的元素变成队列头的元素，这样就变成了先进后出的栈。

20. 有效的括号 - 力扣（LeetCode）

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串 s ，判断字符串是否有效。

有效字符串需满足：

左括号必须用相同类型的右括号闭合。
左括号必须以正确的顺序闭合。
每个右括号都有一个对应的相同类型的左括号。

这题很简单，就是使用一个栈存3种左括号，然后遇到相同左括号的时候弹出。当最后遍历完字符串时如果栈为空则说明括号匹配成功。

这里要考虑的是什么时候匹配失败，其实分为三种情况：

  • 字符串的长度不为偶数，那就一定不能匹配了。这是一种剪枝的思想。
  • 如果匹配到右括号时，栈为空，说明前面完全就没有左括号。
  • 匹配到右括号时，发现栈顶的不是对应的右括号，这就说明当前括号无法匹配。

150. 逆波兰表达式求值 - 力扣（LeetCode）

根据 逆波兰表示法，求表达式的值。

有效的算符包括 +、-、*、/ 。每个运算对象可以是整数，也可以是另一个逆波兰表达式。

注意 两个整数之间的除法只保留整数部分。

可以保证给定的逆波兰表达式总是有效的。换句话说，表达式总会得出有效数值且不存在除数为 0 的情况。

逆波兰表达式：

逆波兰表达式是一种后缀表达式，所谓后缀就是指算符写在后面。

平常使用的算式则是一种中缀表达式，如 ( 1 + 2 ) * ( 3 + 4 ) 。
该算式的逆波兰表达式写法为 ( ( 1 2 + ) ( 3 4 + ) * ) 。
逆波兰表达式主要有以下两个优点：

去掉括号后表达式无歧义，上式即便写成 1 2 + 3 4 + * 也可以依据次序计算出正确结果。
适合用栈操作运算：遇到数字则入栈；遇到算符则取出栈顶两个数字进行计算，并将结果压入栈中

示例1：

这题是经典的栈的应用，逆波兰式其实就是将正常的运算二叉树以后序顺序读出来，它的优势是可以不用括号就实现运算的先后顺序。这题用一个栈存数，当读到运算符号的时候从栈中提出两个数字进行运算再push入栈中即可。

347. 前 K 个高频元素 - 力扣（LeetCode）

给你一个整数数组 nums 和一个整数 k ，请你返回其中出现频率前 k 高的元素。你可以按 任意顺序 返回答案。

这是一个很有意思的题目，要求统计出现次数最多的k个元素。这很好想到使用map计算每个元素出现的次数，但是难点在于如何实现对这个map的排序，尤其是在这题条件下的对value值的排序。众所周知的，go中没有实现对map排序的相关方法。这里给出对map进行根据key排序和value排序的方法。

有了这两种排序方法之后，我们就可以很方便的解决这道题了，首先获取每个元素出现的次数统计countMap，然后根据统计次数即对map进行排序，最后返回排序完的前k个元素即可。

239. 滑动窗口最大值 - 力扣（LeetCode）

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

返回 滑动窗口中的最大值 。

示例1：

输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置 最大值

[1 3 -1] -3 5 3 6 7 3
1 [3 -1 -3] 5 3 6 7 3
1 3 [-1 -3 5] 3 6 7 5
1 3 -1 [-3 5 3] 6 7 5
1 3 -1 -3 [5 3 6] 7 6
1 3 -1 -3 5 [3 6 7] 7

经典hard难度的题目。在一个给定的窗口中，得出一个最大值，并且这个窗口是会移动的，很容易想到用大顶堆去做这个题目。每次移动窗口，都是将窗口右边的新元素加入大顶堆中，但是窗口左边的元素是一件非常难处理的事。因为大顶堆的堆顶永远都是最大的元素，每次从堆顶去除的元素只能是最大的元素，而不能每次准确的去除窗口最左边出窗口的那个元素。这里我看了题解，才想到可以用数组下标作为堆中的元素。只要把排序规则改成根据下标去数组中获取元素就可以了。然后每次窗口移动的时候，判断堆顶元素即最大元素的坐标，如果最大元素的坐标在窗口左边以外，则说明可以去除这个元素了。

  • 入队列：如果入队元素
